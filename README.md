# Análise de Acessibilidade

O protótipo foi desenvolvido pensando principalmente em usuários idosos ou pessoas em situação de mal-estar físico.

Para melhorar a acessibilidade e a usabilidade foram utilizadas:

- Fontes maiores e legíveis
- Botões com áreas de clique amplas
- Poucos elementos por tela
- Informações organizadas de forma clara
- Hierarquia visual simples
- Ícones fáceis de reconhecer
- Alto destaque para ações importantes

As telas evitam excesso de informações para reduzir a carga cognitiva do usuário em momentos de emergência ou desconforto.

# Fluxo Crítico

## Caminho para iniciar uma consulta de urgência

1. O usuário acessa a tela inicial do aplicativo.
2. Seleciona a opção “Consulta Agora”.
3. Inicia a triagem rápida informando os sintomas.
4. O sistema direciona o usuário para a fila virtual.
5. A tela de espera informa:
   - quantidade de pacientes na frente
   - tempo estimado
6. Quando o médico entra na sala, a videochamada é iniciada automaticamente.

# Prevenção de Erros

Para evitar erros durante a utilização do aplicativo foram aplicadas algumas soluções:

- O botão de emergência permanece sempre visível.
- O botão de encerrar consulta possui confirmação antes de fechar a chamada.
- Informações importantes aparecem destacadas visualmente.
- Os botões principais possuem tamanho maior para evitar toques incorretos.
- Durante a fila de espera o sistema informa claramente o status da consulta.
- A interface foi simplificada para reduzir confusão durante o uso.
